Project Overview

This portfolio page connects the original VET/VTO warehouse forecasting application with an advanced multi-agent Agentic AI version. The system uses historical demand data, machine learning, scenario validation, risk checks, cost analysis, and AI-assisted decision workflows to support real-world warehouse labor planning decisions.

Problem Forecast weekly warehouse workload and reduce staffing uncertainty.
Decision Output Recommend VET, VTO, or Normal staffing actions based on forecasted demand.
Business Goal Support labor planning, overtime control, and operational efficiency.

Project Highlights

  • XGBoost-based weekly workload forecasting
  • VET/VTO/Normal staffing recommendation logic
  • Model comparison against baseline forecasting methods
  • Labor cost impact and potential staffing savings analysis
  • Streamlit dashboard and Flask API architecture
  • Docker containerization for frontend and backend services
  • AWS ECS Fargate deployment evidence
  • Clean portfolio-ready GitHub structure

Business Value

Warehouse labor planning often faces demand volatility, overtime pressure, and mismatches between available labor and operational workload. This project shows how forecasting and decision automation can help operations teams anticipate staffing needs earlier, reduce uncertainty, and support more data-driven VET/VTO decisions.

Technical Architecture

The system combines a machine learning forecasting workflow, a Flask API backend, and a Streamlit dashboard frontend. The application was containerized using Docker and structured for cloud deployment, including AWS ECS Fargate deployment evidence.

  • Forecasting: Python, XGBoost, time-series features, baseline comparison
  • Backend: Flask API for forecast and staffing recommendation logic
  • Frontend: Streamlit dashboard for scenario planning and visualization
  • Deployment: Docker, GitHub, Render, and AWS ECS Fargate evidence
